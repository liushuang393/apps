# LAMS テスト観点（実業務で使えるか）

**原則**: 「画面が出る」「接続を試みた」では合格にしない。**実際に業務として成立すること**＝
**表示（正しい字幕/翻訳が見える）** と **動作（音声が聞こえる・操作が効く）** の両方を満たして初めて合格。

凡例: ✅=実機/実APIで検証済 / ⏳=未検証（実機 or 自動ハーネス要） / ⛔=既知の不可

## B. mode2（ASR→翻訳→TTS）＝明日のリリース対象

| # | 観点 | 合否基準（表示/動作） | 状態 |
|---|------|----------------------|------|

| B2 | **字幕表示(実ループ)** | 話者が発話→**聞き手の画面に正しい翻訳字幕が出る**（原文でなく訳文、話者名つき、数秒以内） | ⏳ 要実機/ハーネス |

| B6 | 多言語同時 | ja発話を en/zh の聞き手が各自の言語で同時に字幕/音声受信 | ⏳ |

## C. mode1（リアルタイムS2S）＝リリース後に再設計

| # | 観点 | 合否基準（表示/動作） | 状態 |
|---|------|----------------------|------|
| C1 | provider配線 | realtime providerが正しいエンドポイント/モデルで接続成功 | ⛔ 下記の通り両経路ともブロック |

#### mode1 再設計の調査結果（2026-06-24）
- **`gemini_live`（推奨・最速）**: プロバイダー実装・モデル(`gemini-3.5-live-translate-preview`)・google-genai は準備済み。
  **唯一のブロッカー = `GEMINI_API_KEY` 期限切れ**（実APIで `1007 API key expired` 確認）。新しい鍵を `.env` に入れ
  `AI_PROVIDER=gemini_live` にするだけで動作見込み（コード変更不要）。
- **`gpt_realtime`（要コード改修）**: OpenAIキーは有効でWS接続も確立するが、プロバイダーが**旧 Realtime Beta API 形式**
  （`OpenAI-Beta: realtime=v1`）を使用。OpenAIが廃止済み（`beta_api_shape_disabled`: "Realtime Beta API is no longer
  supported. Please use /v1/realtime for the GA"）。→ `gpt_realtime.py` を GA Realtime API 形式へ書き換えが必要。
  `.env` の `OPENAI_REALTIME_MODEL=models/gemini-3.5-...`（Geminiパス）も OpenAI realtime モデルへ要修正。
- **結論**: 明日のリリースは mode2 で確定。mode1 は「新Gemini鍵」または「gpt_realtime の GA 改修」のどちらかを選んで着手。
| C2 | S2S音声 | 発話→低遅延(<1.5s)で翻訳音声が再生される | ⏳ |
| C3 | S2S字幕 | S2Sと同時に字幕(transcript)も表示される | ⏳ |
| C4 | フォールバック | S2S不可時に mode2(ASR→翻訳→TTS)へ縮退し、無音にならない | ⏳ |
| C5 | mode2との切替 | グローバル切替(AI_PROVIDER)で mode1/mode2 が選べ、各々上記を満たす | ⏳ |

## 検証手段の優先順位
1. **実APIパイプライン検証**（B1）: コンテナ内Pythonで provider を直接叩く。✅実施済。
2. **LiveKit実ループ・ハーネス**（B2/B3/B4/B6）: Python LiveKitクライアントで「話者(WAV publish)」と
   「聞き手(subscribe)」を立て、agent経由で字幕データ＋翻訳音声トラックが届くことを検証。WSLから
   `ws://livekit:7880`(内部)で到達可能。**画面に依存せず実ループを確認できる最有力手段**。
3. **実機ブラウザ**（B2/B3/B9/B10, C系）: Windows本体(localhost)＋他LANマシン(Chromeフラグ)で人が発話。
   WSLのPlaywrightは 192.168.x へ到達できず不可（実機必須）。

## 既知の環境制約
- WSLのPlaywright/cURLは Windows LAN IP(192.168.210.31) に到達不可（`localhost`は可）。
  → ブラウザ実ループ検証は実機で行う。
